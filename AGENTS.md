# AGENTS.md

## 목적
이 저장소의 에이전트는 요청 범위에 맞는 최소 변경을 수행하고, 코드와 문서를 항상 동기화한다.

## 우선순위
1. 명시된 사용자 요청
2. 이 문서
3. `docs/specs/current/*`의 현재 명세
4. `docs/decisions/*`
5. 기존 코드 패턴

## 작업 원칙
- 수정 전에 관련 파일을 먼저 읽는다.
- 범위를 임의로 넓히지 않는다.
- 큰 재작성보다 작고 되돌리기 쉬운 변경을 우선한다.
- `프론트엔드 / 서버 프록시 / 공급자 인터페이스` 분리를 유지한다.
- 공급자 비밀값은 서버에만 둔다. API 키를 프론트엔드에 노출하지 않는다.
- 현재 저장소의 Vue 3, Vite, Pinia, Vuetify, Nitro, TypeScript 패턴을 우선 따른다.

## 언어 규칙
- 운영 문서, 명세 문서, ADR, 변경 이력, 작업 기록은 한국어로 작성한다.
- 사용자에게 보내는 작업 보고도 한국어로 작성한다.
- 코드 식별자, 경로, 라이브러리명, API 이름, 환경 변수 이름은 원문을 유지할 수 있다.

## 테스트 규칙
- 단위 및 통합 검증은 `vitest`를 사용한다.
- 브라우저 E2E는 `playwright`를 사용한다.
- 동작이 바뀌면 관련 테스트를 추가하거나 수정한다.
- 테스트를 실행하지 못했으면 이유와 대체 검증 방법을 함께 보고한다.

## 문서 규칙
- 기대 동작이나 계약이 바뀌면 `docs/specs/current/*`를 갱신한다.
- 구조 결정이나 트레이드오프가 바뀌면 `docs/decisions/*`를 갱신한다.
- 의미 있는 코드 또는 운영 변경은 `docs/changelog/*`에 기록한다.
- 세션 인수인계 맥락이 중요하면 `docs/worklog/*`를 사용한다.

## 변경 통제
- 관련 없는 파일은 수정하지 않는다.
- 요청 범위에 꼭 필요하지 않으면 의존성을 추가하지 않는다.
- 넓은 리팩터링이 필요하면 이유를 남긴다.
- 작업 종료 전에 코드와 문서가 서로 어긋나지 않는지 확인한다.

## 완료 보고 형식
- 요약
- 변경 파일
- 동작 변화
- 테스트
- 갱신한 문서
- 남은 리스크 / 후속 작업
